Localino Live - implementation of LocalinoRemoteApi via Localino REST API (https://api.localino.app).

## Features

Loads localization asset based on current or preferred locale.\
Formatted as Json/Map. Handles single strings, maps and lists, plurals and simple formatting.\

## Getting started

Add library to your pubspec.yaml under dev_dependencies.

```yaml
dev_dependencies:
  build_runner: 2.4.0
  localino_builder: 0.1.0
```

Add `build.yaml` to your project files to configure builder and localino project.

```yaml
targets:
  $default:
    builders:
      localino_builder:
        options:
          space: 'your space id'
          project: 'your project id'
          access: 'your access token'
```


## Usage

Just run `build_runner` to download your translations into assets folder.

```
dart run build_runner build
```

## Typed key generation (opt-in)

Beyond downloading JSON, the builder can emit a Dart file of typed key
constants so call sites get compile-time safety, autocomplete, and hover docs
instead of raw magic strings.

Enable it with the `generate_keys` option:

```yaml
targets:
  $default:
    builders:
      localino_builder:
        options:
          space: 'your space id'
          project: 'your project id'
          access: 'your access token'
          generate_keys: true    # default false — opt-in, non-breaking
          report_unused: true     # optional — list keys with no LocalinoKeys.* ref
```

Generation runs on the **standalone** path the package already uses to download
JSON — it is not a `build_runner build` source builder. Trigger it directly:

```
dart run localino_builder --generate-keys --report-unused
```

Output is written to `lib/generated/localino_keys.dart`:

```dart
// GENERATED by localino_builder — do not edit by hand.
abstract final class LocalinoKeys {
  /// Add
  static const action_add = 'action_add';

  /// Version: {version} ({number})
  /// params: version, number
  static const version = 'version';

  /// Delete locale
  /// ⚠ missing in: cs_CZ
  static const action_delete_locale = 'action_delete_locale';
}
```

Call sites become:

```dart
title: Localize.i(LocalinoKeys.action_add),
```

The const **value** is always the original JSON key verbatim — runtime lookups
are unchanged. Doc comments show a default-locale preview, `params:` for format
strings, and a `⚠ missing in:` note for keys absent in other locales.

With `report_unused: true`, after generation any key with no
`LocalinoKeys.<name>` reference under `lib/` is printed (report-only, never
deleted) — useful for pruning dead keys once call sites are migrated.
